# Команда /chat - Перегляд переписки між користувачами

## Опис

Команда `/chat` дозволяє адміністратору переглянути повну переписку між двома конкретними користувачами в форматі діалогу.

## Синтаксис

```
/chat ID1 ID2 [кількість]
```

**Параметри:**
- `ID1` - Chat ID першого користувача
- `ID2` - Chat ID другого користувача
- `[кількість]` - (опціонально) скільки повідомлень показати (за замовчуванням 50, максимум 200)

## Приклади використання

### Базове використання
```
/chat 123456789 987654321
```
Показує останні 50 повідомлень між користувачами `123456789` та `987654321`

### З вказаною кількістю
```
/chat 123456789 987654321 100
```
Показує останні 100 повідомлень між цими користувачами

### З максимальною кількістю
```
/chat 123456789 987654321 200
```
Показує останні 200 повідомлень (максимум)

👥 **user2** (17.07 14:40)

👤 **user1** (17.07 14:45)

## Легенда

- **👤** - повідомлення від першого користувача (ID1)
- **👥** - повідомлення від другого користувача (ID2)
- **📋** - тип повідомлення (Text, Photo, Voice, Video, Audio, Document, Sticker, VideoNote)
- **💬** - текстовий вміст (якщо є)

## Відмінності від /history

| Команда | Призначення | Формат |
|---------|-------------|--------|
| `/history ID` | Показує всю історію одного користувача (з усіма співрозмовниками) | Хронологічний список з позначкою напрямку (➡️/⬅️) |
| `/chat ID1 ID2` | Показує тільки переписку між двома конкретними користувачами | Діалог з іменами відправників |

## Типові випадки використання

### 1. Перевірка активного діалогу
Перед з'єднанням користувачів або після їх роз'єднання:
```
/users  # Подивитись, хто з ким спілкується
/chat 123456789 987654321  # Переглянути їх переписку
```

### 2. Модерація
Перевірка скарг на порушення:
```
/chat 123456789 111222333 100  # Переглянути останні 100 повідомлень
```

### 3. Аналіз спілкування
```
/chat 123456789 987654321 200  # Максимальна історія
```

## Обмеження

### Ліміт повідомлень
- За замовчуванням: **50 повідомлень**
- Максимум: **200 повідомлень**
- Якщо потрібно більше, використовуйте SQL запити безпосередньо до БД

### Ліміт Telegram
- Telegram обмежує повідомлення **4096 символами**
- Якщо переписка занадто довга, повідомлення буде обрізано
- Рекомендація: використовувати меншу кількість повідомлень або експорт в файл

### Продуктивність
Для великих історій (100-200 повідомлень) запит може виконуватись 1-3 секунди.

## SQL альтернатива

Для більш детального аналізу можна використовувати SQL:

```sql
-- Вся переписка без обмежень
SELECT 
    mh.SentAt,
    CASE WHEN mh.FromChatId = 123456789 THEN 'User1' ELSE 'User2' END AS Sender,
    mh.MessageType,
    mh.TextContent
FROM MessageHistories mh
WHERE 
    (mh.FromChatId = 123456789 AND mh.ToChatId = 987654321)
    OR
    (mh.FromChatId = 987654321 AND mh.ToChatId = 123456789)
ORDER BY mh.SentAt DESC;
```

## Код реалізації

Команда використовує метод `MessageHistoryService.GetMessageHistoryBetweenUsersAsync()`:

```csharp
var chatHistory = await _messageHistoryService.GetMessageHistoryBetweenUsersAsync(
    chatId1: user1Id,
    chatId2: user2Id,
    count: 50
);
```

Цей метод виконує SQL запит:
```sql
SELECT * FROM MessageHistories
WHERE 
    (FromChatId = @chatId1 AND ToChatId = @chatId2)
    OR
    (FromChatId = @chatId2 AND ToChatId = @chatId1)
ORDER BY SentAt DESC
TAKE @count
```

## Поради

1. **Спочатку /users** - подивіться список користувачів та їх статуси
2. **Починайте з меншої кількості** - `/chat ID1 ID2` (50 повідомлень)
3. **Збільшуйте за потреби** - `/chat ID1 ID2 100`
4. **Для великих історій** - використовуйте SQL або експорт

## Майбутні покращення

Потенційні додаткові функції:
- `/chat ID1 ID2 export` - експорт переписки у файл (JSON/CSV/TXT)
- `/chat ID1 ID2 search "keyword"` - пошук по ключовим словам
- `/chat ID1 ID2 from 01.01.2024 to 31.01.2024` - переписка за період
- `/chat ID1 ID2 stats` - статистика переписки (кількість повідомлень, типи і т.д.)
