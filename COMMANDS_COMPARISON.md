# Візуальне порівняння команд /history та /chat

## Структура даних в базі

```
MessageHistories (таблиця)
┌────┬────────────┬──────────┬─────────────┬──────────────┐
│ Id │ FromChatId │ ToChatId │ MessageType │  TextContent │
├────┼────────────┼──────────┼─────────────┼──────────────┤
│ 1  │ 123        │ 456      │ Text        │ "Привіт!"    │
│ 2  │ 456        │ 123      │ Text        │ "Як справи?" │
│ 3  │ 123        │ 789      │ Photo       │ null         │
│ 4  │ 789        │ 123      │ Voice       │ null         │
│ 5  │ 123        │ 456      │ Video       │ "Подивись"   │
└────┴────────────┴──────────┴─────────────┴──────────────┘
```

---

## Команда: /history 123

**SQL запит (спрощено):**
```sql
SELECT * FROM MessageHistories
WHERE FromChatId = 123 OR ToChatId = 123
ORDER BY SentAt DESC
LIMIT 20
```

**Результат:**
```
📜 Історія повідомлень користувача `123`:

➡️ 14:30 | Text      (123 → 456)
   Привіт!

⬅️ 14:32 | Text      (456 → 123)
   Як справи?

➡️ 14:35 | Photo     (123 → 789)  ← повідомлення іншому користувачу!

⬅️ 14:37 | Voice     (789 → 123)  ← від іншого користувача!

➡️ 14:40 | Video     (123 → 456)
   Подивись
```

**Що показується:**
- ✅ Всі повідомлення відправлені користувачем 123
- ✅ Всі повідомлення отримані користувачем 123
- ✅ Спілкування з різними партнерами (456, 789)

---

## Команда: /chat 123 456

**SQL запит (спрощено):**
```sql
SELECT * FROM MessageHistories
WHERE (FromChatId = 123 AND ToChatId = 456)
   OR (FromChatId = 456 AND ToChatId = 123)
ORDER BY SentAt DESC
LIMIT 50
```

**Результат:**
```
💬 Переписка між `123` та `456`:

👤 123 (14:30)
   📋 Text
   💬 Привіт!

👥 456 (14:32)
   📋 Text
   💬 Як справи?

👤 123 (14:40)
   📋 Video
   💬 Подивись
```

**Що показується:**
- ✅ Тільки повідомлення між 123 та 456
- ❌ НЕ показує повідомлення між 123 та 789

---

## Візуальна схема

### Користувач 123 спілкується з двома людьми:

```
       456 ←→ 123 ←→ 789
         ↑      ↑      ↑
    Партнер 1  User  Партнер 2
```

### /history 123 (повна історія)

```
┌─────────────────────────────────┐
│   Вся історія користувача 123   │
├─────────────────────────────────┤
│  📨 123 → 456: "Привіт!"        │
│  📨 456 → 123: "Як справи?"     │
│  📨 123 → 789: [Photo]          │  ← Інший партнер!
│  📨 789 → 123: [Voice]          │  ← Інший партнер!
│  📨 123 → 456: [Video]          │
└─────────────────────────────────┘
```

### /chat 123 456 (діалог з конкретним партнером)

```
┌─────────────────────────────────┐
│   Діалог: 123 ↔ 456             │
├─────────────────────────────────┤
│  📨 123 → 456: "Привіт!"        │
│  📨 456 → 123: "Як справи?"     │
│  📨 123 → 456: [Video]          │
└─────────────────────────────────┘
       ⚠️ Повідомлення до 789 НЕ показані
```

## Практичні приклади

### Приклад 1: Користувач спілкувався з трьома людьми

**База даних:**
```
User 100 → User 200: "Привіт, User 200!"
User 200 → User 100: "Привіт!"
User 100 → User 300: "Привіт, User 300!"
User 300 → User 100: "Вітаю!"
User 100 → User 400: "Привіт, User 400!"
User 400 → User 100: "Здоров!"
```

**Команда: `/history 100`**
```
📜 Історія користувача 100:
➡️ → 200: "Привіт, User 200!"
⬅️ ← 200: "Привіт!"
➡️ → 300: "Привіт, User 300!"
⬅️ ← 300: "Вітаю!"
➡️ → 400: "Привіт, User 400!"
⬅️ ← 400: "Здоров!"
```
✅ Показує ВСЕ

**Команда: `/chat 100 200`**
```
💬 Діалог 100 ↔ 200:
👤 100: "Привіт, User 200!"
👥 200: "Привіт!"
```
✅ Показує ТІЛЬКИ діалог з User 200

---

### Приклад 2: Конфлікт між користувачами

**Ситуація:**
User 555 скаржиться на User 666.

**Дії адміністратора:**

1. **Перевірити загальну активність User 666:**
```
/history 666 50
```
Результат: бачимо всю його активність

2. **Перевірити конкретний діалог:**
```
/chat 555 666 100
```
Результат: бачимо тільки їх переписку

3. **Якщо підтвердилося порушення:**
```
/disconnect 666
```

## Коли використовувати що?

### Використовуйте `/history ID`

✅ Коли потрібно:
- Проаналізувати загальну активність користувача
- Побачити з ким він спілкується
- Знайти підозрілу поведінку
- Отримати статистику користувача

❌ НЕ використовуйте коли:
- Потрібен конкретний діалог
- Занадто багато партнерів (вивід буде заплутаним)

### Використовуйте `/chat ID1 ID2`

✅ Коли потрібно:
- Прочитати конкретну розмову
- Перевірити скаргу на діалог
- Експортувати переписку
- Зрозуміти контекст конфлікту

❌ НЕ використовуйте коли:
- Не знаєте другого користувача
- Потрібна повна картина активності

## SQL еквіваленти (для розробників)

### /history ID
```sql
-- Вся історія користувача
SELECT 
    CASE WHEN FromChatId = @UserId THEN '➡️' ELSE '⬅️' END AS Direction,
    CASE WHEN FromChatId = @UserId THEN ToChatId ELSE FromChatId END AS Partner,
    MessageType,
    TextContent,
    SentAt
FROM MessageHistories
WHERE FromChatId = @UserId OR ToChatId = @UserId
ORDER BY SentAt DESC
LIMIT @Count;
```

### /chat ID1 ID2
```sql
-- Діалог між двома користувачами
SELECT 
    FromChatId AS Sender,
    MessageType,
    TextContent,
    SentAt
FROM MessageHistories
WHERE 
    (FromChatId = @User1 AND ToChatId = @User2)
    OR
    (FromChatId = @User2 AND ToChatId = @User1)
ORDER BY SentAt DESC
LIMIT @Count;
```
