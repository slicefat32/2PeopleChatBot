# Робота з історією повідомлень

## Огляд

Бот автоматично зберігає всі повідомлення, які проходять через нього, у базу даних SQL Server. Це дозволяє:
- Відслідковувати всю комунікацію між користувачами
- Переглядати історію спілкування
- Аналізувати типи повідомлень
- Зберігати докази спілкування

## Модель MessageHistory

```csharp
public class MessageHistory
{
    public long Id { get; set; }                  // Унікальний ID запису
    public long FromChatId { get; set; }          // Відправник
    public long ToChatId { get; set; }            // Отримувач
    public int MessageId { get; set; }            // ID повідомлення в Telegram
    public string MessageType { get; set; }       // Тип повідомлення
    public string? TextContent { get; set; }      // Текст повідомлення (якщо є)
    public string? FileId { get; set; }           // File ID (для медіа)
    public DateTime SentAt { get; set; }          // Час відправки (UTC)
}
```

## Типи повідомлень

Бот розпізнає та зберігає наступні типи:

| MessageType | Опис | TextContent | FileId |
|-------------|------|-------------|--------|
| `Text` | Текстове повідомлення | Текст повідомлення | - |
| `Photo` | Фотографія | Підпис (якщо є) | FileId останнього фото |
| `Voice` | Голосове повідомлення | - | FileId голосового |
| `Audio` | Аудіофайл | Підпис (якщо є) | FileId аудіо |
| `Video` | Відеофайл | Підпис (якщо є) | FileId відео |
| `Document` | Документ | Підпис (якщо є) | FileId документу |
| `Sticker` | Стікер | - | FileId стікера |
| `VideoNote` | Кружечок | - | FileId відео |

## Команди адміністратора

### `/history ID [кількість]`

Переглянути історію повідомлень користувача.

**Параметри:**
- `ID` - Chat ID користувача
- `[кількість]` - кількість повідомлень для показу (опціонально, за замовчуванням 20, максимум 100)

**Приклади:**
```
/history 123456789
```
Покаже останні 20 повідомлень користувача 123456789

```
/history 123456789 50
```
Покаже останні 50 повідомлень користувача 123456789

**Формат виводу:**
```
📜 Історія повідомлень користувача `123456789` (останні 20):

➡️ 17.07.2026 14:30 | Text
   Привіт! Як справи?...

⬅️ 17.07.2026 14:32 | Photo
   Подивись на це фото!...

➡️ 17.07.2026 14:35 | Voice

⬅️ 17.07.2026 14:40 | Sticker
```

**Позначення:**
- ➡️ - повідомлення ВІДПРАВЛЕНЕ користувачем
- ⬅️ - повідомлення ОТРИМАНЕ користувачем

### `/chat ID1 ID2 [кількість]`

Переглянути переписку між двома конкретними користувачами.

**Параметри:**
- `ID1` - Chat ID першого користувача
- `ID2` - Chat ID другого користувача
- `[кількість]` - кількість повідомлень для показу (опціонально, за замовчуванням 50, максимум 200)

**Приклади:**
```
/chat 123456789 987654321
```
Покаже останні 50 повідомлень між користувачами

```
/chat 123456789 987654321 100
```
Покаже останні 100 повідомлень між користувачами

**Формат виводу:**
```
💬 Переписка між `user1` та `user2` (останні 50):

👤 **user1** (17.07 14:30)
   📋 Text
   💬 _Привіт! Як справи?_

👥 **user2** (17.07 14:32)
   📋 Photo
   💬 _Подивись на це фото!_

👤 **user1** (17.07 14:35)
   📋 Voice

👥 **user2** (17.07 14:40)
   📋 Sticker
```

**Позначення:**
- 👤 - перший користувач (ID1)
- 👥 - другий користувач (ID2)

## MessageHistoryService API

### SaveMessageAsync
```csharp
await _messageHistoryService.SaveMessageAsync(
    fromChatId: 123456789,
    toChatId: 987654321,
    messageId: 12345,
    messageType: "Text",
    textContent: "Привіт!",
    fileId: null
);
```

### GetMessageHistoryBetweenUsersAsync
```csharp
var history = await _messageHistoryService.GetMessageHistoryBetweenUsersAsync(
    chatId1: 123456789,
    chatId2: 987654321,
    count: 50
);
```
Отримує історію між двома конкретними користувачами.

### GetUserMessageHistoryAsync
```csharp
var history = await _messageHistoryService.GetUserMessageHistoryAsync(
    chatId: 123456789,
    count: 50
);
```
Отримує всю історію користувача (відправлені + отримані).

### GetTotalMessagesCountAsync
```csharp
int total = await _messageHistoryService.GetTotalMessagesCountAsync(
    chatId: 123456789
);
```
Підраховує загальну кількість повідомлень користувача.

## Індекси бази даних

Для оптимізації пошуку створені індекси на:
- `FromChatId` - швидкий пошук по відправнику
- `ToChatId` - швидкий пошук по отримувачу
- `SentAt` - сортування за часом

## Приватність і GDPR

⚠️ **Важливо:** Зберігання історії повідомлень може підпадати під регулювання GDPR та інші закони про захист даних.

Рекомендації:
1. Інформуйте користувачів про збереження їх повідомлень
2. Реалізуйте можливість видалення історії на вимогу користувача
3. Захистіть базу даних належним чином
4. Обмежте доступ до історії тільки авторизованим адміністраторам
5. Розгляньте можливість автоматичного видалення старих повідомлень

## Розширення функціоналу
